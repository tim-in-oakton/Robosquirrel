Robosquirrel, adventures in rodent design patterns (WIP)

I love chipmunks, they are loveable little rodents that collect the birdseed under our feeder and occasionally chew through our garage screens to get at the bags of birdseed.  They live in harmony with the neighborhood birds and I sometimes reward their coolness with manna from heaven (unsalted peanuts that I toss off the deck to them).

Cool:

Now squirrels on the other hand I like less.  Sure, they are territorial acrobatic morons that are fun to watch, but they chase the birds from our feeder and usually empty the feeder to get the sunflower seeds.  The chipmunks love this, but the birds either go hungry or tempt fate by eating at cat level after the feeder is dumped out.

Not Cool:

/* picture of little bastard dumping out my feeder */

At happy hour with some fellow enginerds, we discussed different approaches to tree-rat mitigation with the following constraints:

  • No harm to the rodents.  Wifey was a vet tech, kids are both Dr Doolittles, no harm to the squirrels will be tolerated.
  • Has to be automated, should not scare off birds ( low false positive rate).
  • Simple, scalable and replicable - minimize undifferentiated heavy lifting, keep it simple stupid, leverage COTS stuff where possible.

Later in the happy hour (2 beers later to be specific) we had a base config and high level roadmap -

  • Raspberry Pi because it’s cheap, plentiful in my workshop and because everybody likes Pi (ideally Pi0W, assuming that it can power a decent webcam)
  • Picam to start (cheap and well integrated, maybe use security cam later?)
  • Decoupled squirrel mitigation hardware - use a simple (BLE?  Web-enabled outlet?  Dry relay?) interface to drive current to a variety of squirrel deterring things until the best one is found (puff of air, squirt water, bright light, sound, close the feeder door, etc).
  • Start by using cloud for image tagging/squirrel classification, get it working.  Later move to building a corpus of squirrel pics, training a portable squirrel model, run that in cloud function or on the Pi.  Who knows - use TPU at the edge?
  • Disclosure, although I work for Google, their AIML APIs are wonderful, accessible and simple so I would have picked that platform regardless.  Google’s cloud is going to power this squirrel project.

OK, what is the pseudocode for the edge device in V1.0?

Loop

        Wait for motion to be detected on lowres image

        Take a highres image

        if(Google Cloud image classifier API sees a squirrel) THEN

                Do something annoying to squirrel (trigger output)

                Take photoroll of annoyed squirrel pics for 3 secs

Until (wine is better than beer)

That’s pretty much it.  We’ll use Python for the scripting because it’s super simple and all the kids are using it these days - not to mention lots of published snippets on hobby sites & git repos.  We’ll specifically use Google’s Cloud Vision API which will be passed a photo and it will return the weighted probabilities of the things (“tags”) that it sees in the photo.  The Vision API can do a lot more, if you’re not just here for the squirrels you can read more HERE.

STEPS-

Install image for Pi, and add Pi Camera (Google for directions, follow them)

Enable using raspi-conf the following:  ssh, vnc, camera

Log into the Pi using both ssh and VNC from your development system (PC is what I used)

Install the Google SDK for the cloud vision APIs (note no cache directive, Pi0 has little memory, and as I’m using Python 3.X I’m installing/using pip3):

sudo apt-get python3-pip

pip3 install --no-cache-dir --upgrade google-cloud-vision  

(this takes a long time on a pi 0 - literally a couple of hours to recompile - go play with your kids while it compiles).

OK, now that we have your workstation taking to your Pi and all the necessary stuff installed, it’s time to do the cloud work to get this show on the road.  The high level steps to replicate what I did are as follows, you may simplify/omit some steps if you like:

  1. Create a Google Cloud Account, create a project in the account, turn on the Vision API and create a “service account” that is allowed to poke the vision API (first 1000 dips/month free).  Download and SECURE the credentials (private key) that will be used to authenticate your calls to the platform.
  2. Optional - I like git, and it makes multi-device development easier.  I put the code on a git account/repo (https://github.com/tim-in-oakton/Robosquirrel) and used Atom (great editor) on PC’s to do the development.  I’d save, stage, commit and push code revisions up to the repo, and the a simple “git pull” on the Pi to download the updates.  You are free to just throw the Python scripts in a director and run them as you like, but this made the dev process a bit friendlier than vi, command line and a 80*24 terminal.
  3. Sort-of-Optional - there are lots of ways to connect code to the real world, but to make this maximally accessible to non-hardware-geeks I chose to NOT use relays, BLE, optocouples, arduinos and wire to annoy the squirrels - I used some “Smart Life” smart outlets I had laying around and used IFTT to control them.  The process of having the Pi tap a webhook should be similar for most other “smart outlets”.  Set up an IFTT.com account, connect your outlets, set up “Web Hooks” in IFTT, and create two events - turn on the outlet with one message, turn it off with another.  

Most nerds should be able to follow these steps as written, I’ll add links to make it more straightforward when I get around to it.
